## 1. システムの目的（確定）

- 短期〜中期（原則1ヶ月以内に閉じる）の戦略研究・検証基盤
- 長期積立(NISA)とは完全別軸。これは「趣味の研究＋意思決定支援」
- 人間が監視しきれないETF・暗号資産・セクター間の動きを機械的に捉える
- **最重視: 月次AIレビュー**。買い候補より「負けた理由を冷静に裁く」

## 2. 監視ユニバース（確定）

### ETF 6種：セクター分散 × 低相関 × 高流動性で選定

| 銘柄 | セクター/資産クラス | 選定理由 |
|---|---|---|
| **QQQ** | 米テック/グロース | 既存。超高流動性。リスクオンの主役 |
| **XLE** | エネルギー | テックと相関が低い。インフレ・原油高で逆行しやすい |
| **XLV** | ヘルスケア | ディフェンシブ。景気サイクルとの相関が低め |
| **XLF** | 金融 | 金利上昇局面でテック・債券と逆の動き。高流動性 |
| **GLD** | 金（コモディティ） | 既存。株と低〜逆相関。リスクオフの逃避先 |
| **TLT** | 米長期国債 | 株と逆相関しやすい。金利・景気の鏡 |

設計意図: QQQ/XLE/XLV/XLF でセクターを散らし、GLD/TLT で資産クラスを散らす。
→ 相関ヒートマップが「意味のある分散」になり、レジーム判定（リスクオン/オフ）の材料が揃う。

### 暗号資産 2種

| 銘柄 | 取得 |
|---|---|
| **BTC** | yfinance `BTC-USD` |
| **ETH** | yfinance `ETH-USD` |

※ 全銘柄 yfinance 統一。暗号資産と取引所価格の差は許容（シグナル用途のため）。

## 4. クオント戦略群（Claudeおまかせ・実装する定石）

陣内さんから「クオントのノウハウをふんだんに、難しくてOK、おまかせ」と指示。
以下を `strategies` に登録し、月次でバックテスト比較する。AIは①既存の選択・調整 ②新規提案 の両方を行う。

### 4.1 トレンド/モメンタム系
- **デュアルモメンタム**: 絶対モメンタム（自身の過去12ヶ月リターンがプラスか）× 相対モメンタム（ユニバース内リターン上位）。両方満たす銘柄に乗る古典かつ強力な手法。
- **時系列モメンタム(TSMOM)**: 各銘柄の過去リターン符号でロング/退避を切替。
- **MAクロス + ADXフィルタ**: 50/200日クロスに、トレンド強度ADXで「ダマシ」を除外。

### 4.2 ボラティリティ調整系
- **ボラティリティターゲティング**: 各ポジションを「目標年率ボラ（例10%）÷ 実現ボラ」で動的サイジング。荒れた資産は小さく、凪いだ資産は大きく持つ。
- **逆ボラ加重**: 複数ポジション保有時、ボラの逆数で資金配分（リスクパリティの簡易版）。

### 4.3 クロスアセット/レジーム系
- **レジーム判定**: リスクオン/オフを地合い指標（QQQ vs TLT/GLD の相対強度、ボラ水準）で分類し、レジームに応じて戦略を切替。
- **相関ブレイク検知**: BTC/QQQ や XLK/XLU 等の相関急変を捉え、分散効果低下を警告 → ポジション縮小トリガ。

### 4.4 リスク制約（全戦略共通）
- 最大保有30日（時間切れ撤退）
- 戦略ごと最大同時ポジション数
- 損切り・利確ルール必須
- 手数料・スリッページをバックテストに必ず織り込む（楽観バイアス排除）

### 戦略ステータス（昇格/降格の規律）
`RESEARCH` → `PAPER` → `MICRO_AUTO` → `MANUAL_APPROVAL` （不調なら `SUSPENDED` / `RETIRED`）
月次レビューの数値判定でステータスを機械的に動かす。

## 5. 月次AIレビュー（このシステムの心臓）

### 5.1 二段構え：先に数字、後でAI
1. **数値評価（Python自動・月末バッチ）**: 各戦略を `PASS / WATCH / FAIL / INVALID`
   - 勝率・期待値・平均損益・最大ドローダウン・手数料込み損益
   - **QQQ単純保有との比較（ベンチマーク劣後なら容赦なくFAIL寄り）**
   - シャープレシオ・連敗数・リスク調整後リターン
2. **AI考察（Claude月次チャット）**: 数値の「理由」を多面的に
   - 反省役: 勝因・敗因・改善点
   - 反証役: 過剰最適化・偶然勝ちを疑う（少サンプル警告）
   - リスク役: 連敗・最大損失・資金配分リスク
   - **時事考察役: web検索で当月ニュースを収集し、数値の背景を説明**（例: 「XLEの好成績は中東情勢による原油高が主因。再現性は低い」）
   - 集約役: 次月方針・戦略の昇格/降格/停止/新規提案

### 5.2 出力（2系統）
- **① LINE通知**: 月初に要点（戦略別PASS/FAIL・次月方針・警告）を短文で
- **② Markdownレポート**: `docs/reviews/YYYY-MM.md` としてGitHubに自動コミット。全文・数値表・AI考察・ニュース根拠を残す

### 5.3 出力例
```text
【market-pilot 月次レビュー 2026年5月】
■総括: 当月+2.1%（QQQ +3.4% に劣後 → 要改善）
■戦略別:
  ・デュアルモメンタム … PASS（QQQ対比+0.8%、DD小）
  ・MAクロス … FAIL（ダマシ多発、QQQに劣後 → PAPER降格）
■時事: 5月は金利据置観測でテック堅調。XLE弱含み(原油安)
■反証: モメンタムの勝ちは地合い依存の疑い。サンプル30件未満で確証不足
■次月: ボラ上昇局面に備えvol-targetを主軸候補に。XLF監視強化
```

## 6. DB設計（Supabase無料枠・最小から）

無料枠1プロジェクトで運用。最初から全部作らず、使う直前に追加。

### Stage A（記録基盤）: 6テーブル
| テーブル | 役割 |
|---|---|
| `tickers` | 銘柄マスタ（asset_classでETF/暗号区別） |
| `strategy_rules` | クオント戦略の定義・パラメータ |
| `market_snapshots` | 日次の価格・指標スナップ（相関の材料） |
| `signals` | シグナル履歴（戦略別・理由付き） |
| `run_logs` | 実行履歴（成功/失敗/エラー） |
| `notifications` | LINE通知履歴 |

### Stage B（研究基盤）: レビュー稼働時に追加
| テーブル | 役割 |
|---|---|
| `correlation_snapshots` | 相関スナップ（期間別） |
| `backtest_runs` / `backtest_trades` | 自動バックテスト結果 |
| `paper_positions` | 紙トレードの保有・損益 |
| `monthly_metrics` | 月次の戦略別評価数値（PASS/FAIL含む） |
| `monthly_reviews` | AI月次レビュー本文・ニュース根拠 |

※ 無料枠の容量制約に配慮し、`market_snapshots` の生データは一定期間で間引き・集約する運用を後で入れる。
