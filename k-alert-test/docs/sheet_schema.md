# スプレッドシート設計

## 基本情報

- タイトル: `Kアラート・テスト開発`
- 管理先: 可能であれば `yumekango.com` のGoogle Workspace
- 用途: 公式LINEの初回コメント、AI分解結果、不足確認のやり取り、対応状況の記録

## シート

### `アラート`

| 列 | 項目 | 型 | 入力元 | 備考 |
|---|---|---|---|---|
| A | No | number | GAS | 連番 |
| B | 初回コメント内容 | text | LINE | 初回コメントを原文のまま保存 |
| C | いつの事ですか？ | text | AI/追加回答 | 日時、時期、期限など |
| D | どこで起きましたか？ | text | AI/追加回答 | 場所、施設、部屋など |
| E | だれが起こした人ですか？ | text | AI/追加回答 | 対象者、投稿者、関係者など |
| F | だれに対してのことですか？ | text | AI/追加回答 | 対象者、被害者、依頼先、影響を受けた人など |
| G | なにをどのようにしていましたか | text | AI/追加回答 | 起きたこと、状況、方法、経過など |
| H | 緊急度 | text | AI/ルール | 初期候補: `高`, `中`, `低` |
| I | 対応コメント | text | 手動/AI下書き | 後続対応を記録 |
| J | やり取り全文記録 | text | GAS | 初回コメントと追加質問・回答を時系列で保存 |
| K | 備考 | text | 手動/GAS | 補足、エラー、判断根拠 |

### `設定`

| キー | 値 | 備考 |
|---|---|---|
| trigger_word | Kアラート | 公式LINEで開始する文言 |
| urgency_options | 高,中,低 | 緊急度候補 |
| required_fields | いつの事ですか？,どこで起きましたか？,だれが起こした人ですか？,だれに対してのことですか？,なにをどのようにしていましたか | 完了判定に使う項目 |

### LIFF報告フォーム用シート

| 列 | 項目 | 型 | 入力元 | 備考 |
|---|---|---|---|---|
| A | No | number | GAS | 連番 |
| B | 企業名 | text | LIFF | 必須 |
| C | 名前（任意） | text | LIFF | 任意 |
| D | いつの事ですか？ | text | LIFF | 必須 |
| E | どこで起きましたか？ | text | LIFF | 必須 |
| F | だれが起こした人ですか？ | text | LIFF | 必須 |
| G | だれに対してのことですか？ | text | LIFF | 必須 |
| H | なにをどのようにしていましたか | text | LIFF | 必須 |
| I | その他（自由記載） | text | LIFF | 任意 |
| J | 相談受付希望 | text | LIFF | `希望する` / `希望しない` |

## 保存ルール

- 初回コメントは必ず先に保存する
- AI抽出に失敗しても、初回コメント行は残す
- 追加質問と回答は `やり取り全文記録` に追記する
- `いつの事ですか？・どこで起きましたか？・だれが起こした人ですか？・だれに対してのことですか？・なにをどのようにしていましたか` が揃ったら完了扱いにする
- 緊急度とChatWork通知は第2段階で精度調整する
