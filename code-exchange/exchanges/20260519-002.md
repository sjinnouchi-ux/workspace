# ローカルファイル整理：GitHub状態との同期

## 背景・目的
Claude Desktop（Cowork）がGitHub Web UI運用に切り替える前に、
ローカルWriteツールでファイルを作成していた。
その結果、ローカルにGitHub上と差分が生じている可能性がある。
本タスクはその差分を確認し、ローカルをGitHubの状態に正しく合わせることを目的とする。

## 実行環境
- Python: 3.13.3
- 依存パッケージ: なし（gitコマンドのみ）

## 手順

### Step 1: 最新をpull
```bash
cd /Users/satoshijinnouchi/sjinnouchi-ux-workspace
git pull origin main
```

### Step 2: ローカルの差分を確認
```bash
git status
git diff --stat HEAD
```

以下の観点で確認する：
- **Untracked files**：GitHubにないローカル独自ファイル → 削除候補
- **Modified files**：GitHubと内容が異なるファイル → GitHub版を正として`git checkout`で戻す
- **Deleted files**：ローカルで消えているがGitHubにある → `git checkout`で復元

### Step 3: 判断基準

| 状態 | 対処方針 |
|------|----------|
| Untracked（DesktopのWriteで作成したもの） | 削除（`rm`）。GitHub Web UIで作成済みなら重複 |
| Modified（ローカルとGitHubで差分あり） | GitHubが正。`git checkout -- <file>`で戻す |
| Conflict（add/add等） | GitHub版を採用。`git checkout --theirs <file>` → `git add` |
| 正当なローカル変更 | そのままpush |

### Step 4: クリーンアップ実行
```bash
# 例：不要なuntrackedファイルを確認してから削除
git clean -n  # 削除予定ファイルの確認（実行はしない）
git clean -f  # 実際に削除（確認後のみ実行）

# ローカル変更をGitHub版に戻す場合
git checkout -- <対象ファイル>
```

### Step 5: 結果をclaude_log.mdに記録してpush
```bash
git add -A
git commit -m 'chore: ローカルファイル整理・GitHub状態と同期'
git push origin main
```

## 期待する出力・結果
- `git status` がclean（差分なし）になること
- 不要なローカルファイルが削除されていること
- claude_log.mdに整理結果が記録されていること
